# PWA Setup for Afrikad Mobile App

The mobile app has been configured as a Progressive Web App (PWA) using Expo's built-in PWA support.

## What's Configured

✅ **Web App Manifest** - Configured in `app.json` under the `web` section
✅ **PWA Metadata** - App name, description, theme colors, display mode
✅ **Icons** - Uses existing app icons from `assets/` directory
✅ **Service Worker** - Automatically generated by Expo when building for web

## Configuration

The PWA settings are in `app.json`:

```json
"web": {
  "favicon": "./assets/favicon.png",
  "name": "AfriKAD",
  "shortName": "Afrikad",
  "description": "Afrikad - Digital Wallet & Virtual Card",
  "themeColor": "#000000",
  "backgroundColor": "#000000",
  "display": "standalone",
  "startUrl": "/",
  "scope": "/",
  "orientation": "portrait",
  "lang": "en",
  "bundler": "metro"
}
```

## How It Works

Expo automatically:
- Generates `manifest.json` from the `web` configuration
- Creates a service worker for offline support
- Includes PWA meta tags in the HTML
- Uses your app icons for PWA installation

## Testing the PWA

### Development
```bash
npm run web
# or
npx expo start --web
```

Then open the app in your browser. The PWA manifest and service worker will be automatically generated.

### Production Build
```bash
npx expo export:web
```

This creates a static web build in the `web-build/` directory that can be deployed to any static hosting service.

## Installation

Users can install the PWA by:
1. Opening the app in a supported browser (Chrome, Edge, Safari, etc.)
2. Clicking the install prompt or using the browser's install option
3. The app will appear as a standalone application on their device

## Browser Support

PWAs work on:
- ✅ Chrome/Edge (Desktop & Mobile)
- ✅ Safari (iOS 11.3+, macOS)
- ✅ Firefox (Desktop & Mobile)
- ✅ Samsung Internet

## Requirements

- **HTTPS** - Required in production (or localhost for development)
- **Icons** - Already configured using `assets/icon.png` and `assets/favicon.png`
- **Service Worker** - Automatically generated by Expo

## Customization

To customize the PWA:

1. **Theme Colors** - Update `themeColor` and `backgroundColor` in `app.json`
2. **App Name** - Change `name` and `shortName` in the `web` section
3. **Icons** - Replace files in `assets/` directory:
   - `icon.png` - Main app icon (used for PWA)
   - `favicon.png` - Browser favicon
   - `adaptive-icon.png` - Android adaptive icon

## Notes

- The PWA configuration only affects the web build
- Native iOS and Android builds are unaffected
- Service worker is automatically generated - no manual configuration needed
- Expo handles all PWA infrastructure automatically
